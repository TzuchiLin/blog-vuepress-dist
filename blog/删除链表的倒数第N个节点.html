删除链表的倒数第N个节点

kay`s Blog 2020-09-20 13:56:29 链表

给定一个链表，删除链表的倒数第 n 个节点，并且返回链表的头结点。

# 示例

给定一个链表: 1->2->3->4->5, 和 n = 2.

当删除了倒数第二个节点后，链表变为 1->2->3->5.

说明：

给定的 n 保证是有效的。

进阶：

你能尝试使用一趟扫描实现吗？

以下的解法都会首先在设置一个“哑节点”dummyNode，并且dummyNode.next = head，这只要是为了防止极端情况（只有一个节点或者删除的是头节点），这样可以减少逻辑判断。

# 解法一

使用两次循环，第一次获取链表的长度，第二次找到需要删除节点的前一个，改变其后驱即可。

  • 时间复杂度
  • 空间复杂度

# 代码

/**
 * Definition for singly-linked list.
 * public class ListNode {
 *     int val;
 *     ListNode next;
 *     ListNode(int x) { val = x; }
 * }
 */
class Solution {
    public ListNode removeNthFromEnd(ListNode head, int n) {

        // 设置哑节点
        ListNode dummyNode = new ListNode(0);
        dummyNode.next = head;

        // 计算链表长度
        int length = 0;
        ListNode temp = head;
        while(null != temp){
            length ++;
            temp = temp.next;
        }

        // 删除链表的第n个节点
        length = length - n;
        temp = dummyNode;
        while( --length >= 0) {
            temp = temp.next;
        }

        // 删除节点
        temp.next = temp.next.next;

        // 返回头节点
        return dummyNode.next;
        
    }
}

# 解法二

题目中进阶的解法要求使用使用一趟扫描，其实也就是如何能在一次的循环下技能获取链表的长度，也能删除掉元素

可以采取首先将链表中所有的元素放入ArrayList中，由于ArrayList底层使用数组，所以对ArrayList的随机访问时间复杂度是，满足题目中的扫描一趟的条件。

  • 时间复杂度
  • 空间复杂度

# 代码

/**
 * Definition for singly-linked list.
 * public class ListNode {
 *     int val;
 *     ListNode next;
 *     ListNode(int x) { val = x; }
 * }
 */
class Solution {
    public ListNode removeNthFromEnd(ListNode head, int n) {

        // 设置哑节点
        ListNode dummyNode = new ListNode(0);
        dummyNode.next = head;


        List<ListNode> list = new ArrayList<>();
        ListNode temp = dummyNode;

        // 将链表存入ArrayList中
        while(null != temp){
            list.add(temp);
            temp = temp.next;
        }

        // 删除n处的节点，注意现在头部是哑节点，并不是head
        // 因为哑节点的加入list的长度比原链表的长度大1
        int length = list.size();
        list.get(length - n - 1).next = list.get(length - n - 1).next.next;

        // 返回头节点
        return dummyNode.next;

    }
}

# 解法三

解法三实际上是解法一的优化版本，使用两个指针first和指针sencond均指向头部（哑节点），first首先移动到间隔哑节点n处（距离n+1）的位置，保持着该间隔距离同时向后移动first和second，直到first到达了tail.next，由于n的定义就是倒数第几个节点，也就是待删除节点和tail.next的距离，而此时first已经到达了tail.next的位置，所以此时的second已经到达了待删除节点的前一位，然后更新该节点后驱就可以了。

  • 时间复杂度
  • 空间复杂度

描述可能不够直观，就用画图来说明

  • 刚开始时，first和second均指向哑节点
  • fist移动n+1位，不移动n位是因为实际上我们需要操作的节点是待删除节点的前一个节点
  • first和second开始同时移动
  • first移动到空的位置，注意，从开始到结束first和second一直间隔n个节点，此时second的位置就是待删除节点的前一位
  • 删除此时second的后一个节点

# 代码

/**
 * Definition for singly-linked list.
 * public class ListNode {
 *     int val;
 *     ListNode next;
 *     ListNode(int x) { val = x; }
 * }
 */
class Solution {
    public ListNode removeNthFromEnd(ListNode head, int n) {

        // 开始将 first和second全部指向哑节点   
        ListNode dummary = new ListNode(0);
        dummary.next = head;
        ListNode first = dummary;
        ListNode second = dummary;

        // 将first移动n+1次
        for( int i = 1; i <= n+1;i++){
            first = first.next;
        }

        // 同时移动两个节点直到first到达tail.next
        while(null != first){
            first = first.next;
            second = second.next;
        }

        // 删除second之后的节点
        second.next = second.next.next;
        
        // 返回头节点
        return dummary.next;
    }
}
